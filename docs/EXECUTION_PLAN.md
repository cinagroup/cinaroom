# CinaRoom 落地执行计划

> 基于《Multipass自主品牌Web远程管理平台 完整落地执行计划》整理

## 核心原则

- **安全第一**：Cloudflare Token 仅存云端，用户端无任何敏感密钥
- **极简部署**：用户一键安装，无需配置网络/穿透/端口，开箱即用
- **商用合规**：遵循 Multipass GPLv3 协议，Cloudflare 商用条款
- **可规模化**：免费/付费版分层，支持海量用户并发

## 核心风险与规避

| 风险 | 规避方案 |
|------|----------|
| Cloudflare Token 泄露 | Token 仅云端加密存储，用户端零密钥，全程反向连接 |
| Multipass GPLv3 合规 | 标注原版权，公开修改源码，遵循 GPLv3 商用规则 |
| 海量用户并发瓶颈 | 多账号隔离，云端集群部署，付费版扩容，免费版限流 |
| 用户端网络适配 | 纯出站 WebSocket 连接，无需端口映射，自动重连 |
